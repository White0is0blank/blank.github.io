PE文件结构

Ⅰ.含义

任何一个在Windows上运行的可执行文件都要遵循一定的格式，这个格式就是PE文件的结构。比如：exe、dll、部分sys

PE文件在16进制工具内存中打开时，都是以4D 5A开头的，作为可执行文件的一个标记（MZ）

以WPS为例，运行WPS在winhex中以内存的形式打开WPS

①PE文件分节

分节含义

在内存和硬盘中pe文件被分为了一节一节的存放，节间空隙以0来填充

PE结构是分节的原因
  1. 节省硬盘空间（老的编译器）

  • 硬盘对齐：200h
  • 内存对齐：1000h

​ ——硬盘对齐为了增加读写的速度

由前两点即可得知，当exe在内存中加载时会有一个拉伸的效果

即pe文件存在这两种不同的形态：磁盘文件和内存映像

新的编译器（新的硬盘）

对齐一致，减小了运算量

2.节省硬盘空间

由于多开的原因

●可选PE头(大小不确定32/64位不同)

程序在内存中真正的地址：内存镜像基地址+OEP

​ 原因：ImageBase的地址可能被抢占，为了保证被抢占了之后依旧可以运行

CheckSum

用于检查PE文件的完整性,一般exe文件可以使其值为0，在一些内核模式驱动文件及DLL文件中,该值必须是存在且正确的

校验值的计算很简单:
将PE文件两个两个数据进行累加，如果存不下就自然溢出（剩多少就存多少），将计算结果与CheckSum进行比较,不相等则说明文件被修改过或不完整

重点：带‘**’就是

一个exe不是一个PE文件，而是由多个PE文件构成的

一个exe有一个虚拟的独立的4G空间

Ⅳ.FileBuffer/ImageBuffer

①FileBuffer ——> ImageBuffer

②计算ImageBuffer中某一节表的位置到FileBuffer中的位置

计算在ImageBuffer中某一节表中地址对应FileBuffer中的地址

ImageBuffer中的地址为501234，PE文件初位置为50000

其中节表的大致分布如下：

第一步：501234 - 500000 = 1234 //得出现下的偏移地址

第二步：VirtualAdress < 1234 < Misc.VirtualSize //判断目前处于哪一个节区 ——> 得出在第一个节区

第三步：1234 - VirtualAdress = 234 //关于第一个节区的偏移

第四步：PointerToRawData + 234 = 634

E8、E9计算公式
E8：

call指令对应的硬编码，后接上4个字节的地址跳转（此跳转地址并不是我们想要的跳转的地方）

此处的硬编码E8后的地址并不是函数Function后的跳转地址

真正要跳转的地址 = E8这条指令的下一条指令 + X

于是通过该方程式求取X的值：

E8下一条指令的地址为A918D6

实际跳转的地址0A91023

①X = 真正要跳转的地址 - E8这条指令的下一条指令

即：X = 0xA91023 - 0xA918D6 = 0xFFFF F74D

此值恰好是E8后面的地址

注意：由于存储地址的地方都是四个字节，是双字，所以在利用计算器计算的时候需要选中DWORD

②X = 真正要跳转的地址 -（E8当前的地址（拉伸后的地址） + 5）（E8的下一条指令的地址 = E8 + 硬编码所占的字节数）

E9：

与E8同理

  • Characteristics：由于复制的是代码节，则此处属性可以不用修改，如果有属性的改变可以对照相应的表格修改此处的数据

Ⅷ.扩大节-合并节-数据目录

扩大节

增加最后一个节的空间大小，在pe文件的结尾处（即最后一个节区）添加一段新的内存空间。（该方式是为了避免出现，当节表结束后一段的内存中出现重要信息，导致无法直接在原有节表后添加新的节表，同时这种情况之下提头也显得非常的麻烦，可以使用）

1.在最后一个节区添加新的内存空间：SizeOfImage + Ex

2.将最后一个节的SizeOfRawData和VirtualSize修改为N

SizeOfRawData = VirtualSize = N

N = SizeOfRawData / VirtualSize内存对齐后的值 + Ex

3.修改SizeOfImage

SizeOfImage = SizeOfImage + Ex

合并节

——合并节之后文件会变大

1.拉伸到内存

2.将第一个节的内存大小，文件大小改成一样

Max = SizeOfRawData > VirtualSize?SizeOfRawData:VirtualSize

SizeOfRawData = VirtualSize = 最后一个节的VirtualAddress + Max - SizeOfHeaders内存对齐后的大小

3.将第一个节的属性改为包含所有节的属性

4.修改节的数量为1
